### Namespace routes by user

Currently, there is no name spacing of routes by user. As a result, route slugs need to be unique globally, which may lead to potential conflicts and restrict flexibility. A proposed solution is to introduce namespacing by including the user ID or a similar identifier in the route URL. This way, route slugs need only be unique per user, rather than globally.

#### Benefits:
1. This would prevent collisions between different users who might want to use the same slugs.
2. It ensures route organization and scalability for multi-user environments.

#### Implementation Consideration:
- Include user ID or equivalent in the route definition.
- Update relevant sections of the codebase to support scoped routing.
